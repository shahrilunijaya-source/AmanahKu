1 What is Amanahku

A single HR system covering the full employee lifecycle — hire to retire — for several companies under one login.

Core idea

One HR admin can run multiple companies (tenants) from the same account. Each screen is self-teaching: a guide banner and field hints explain every action, so a brand-new HR clerk needs no manual.

Built for Malaysia

Statutory payroll (EPF, SOCSO, EIS, PCB/MTD), NRIC encrypted at rest, bank-file export (Maybank2u Biz & pluggable formats), bilingual EN/BM across every screen.

Stack
Laravel 13 · Blade · Tailwind v4 · MySQL
Auth
Password + 2FA + Passkey (WebAuthn)

4 Process flow

How a request moves through the system — the same approval spine repeats across leave, OT, claims, expenses, loans, shift-swaps.

  1. Sign in Password / 2FA / passkey → land on dashboard.
  2. Pick company Tenant selector sets which org's data you see.
  3. Pick role Persona switcher changes what the dashboard surfaces.
  4. Self-service action Employee submits (leave, claim, timesheet, OT, swap…).
  5. Approval Manager/HR approves or rejects; status updates; audit log records it.
  6. Downstream Approved items feed payroll, calendars, compliance and reports.
  7. Lifecycle Recruitment → Onboarding → Probation → daily HR → Resignation → Offboarding.

6 How to use it

  1. Open localhost:8000 and log in with the demo account above.
  2. Read the guide banner at the top of each screen — every screen explains itself in EN + BM.
  3. Switch company using the tenant selector (top of sidebar) to change which org you're managing.
  4. Switch persona on the dashboard to see the system as Employee / Manager / Management / HR.
  5. Toggle language EN ⇄ BM to confirm both render.
  6. Do a full round-trip: submit something as Employee, switch persona, approve it as Manager/HR.

7 Local test plan

Tick these to confirm the build before pushing to staging. (Automated suite already green: 328 passed.)

Smoke — does it boot

  • Login page loads, demo account signs in.
  • Dashboard renders with no errors; sidebar lists all groups.
  • Open 8–10 random modules — each loads its guide banner + data.
  • Language toggle EN ⇄ BM flips all visible copy.

Multi-tenant & roles

  • Tenant switch Unijaya → Shell → Petron changes org name & data.
  • As Petron / Employee, admin-only screens (Payroll, Admin) are hidden or blocked.
  • As Unijaya / HR, every module is reachable.

Approval round-trips

  • Leave: apply as Employee → approve as Manager → balance/calendar update.
  • Overtime / Claims / Expenses: submit → approve → reject path also works.
  • Shift swap: request → counterpart accept → manager approve.

Payroll (Malaysia)

  • Create a payroll run; EPF/SOCSO/EIS/PCB compute.
  • Finalise → approve → download bank file + statutory report.
  • NRIC shows masked; payslip opens.

Lifecycle & security

  • Recruitment → move candidate stages; Onboarding checklist toggles.
  • Probation check-in & decision; Resignation → exit interview.
  • Enrol a passkey / enable 2FA; audit log records key actions.

Provisioning & onboarding (new)

  • Sign in as super admin → console lists companies.
  • Create a company → first HR admin issued a one-time password (shown + emailed).
  • Log in as that new admin → forced to reset password before reaching the app.
  • Company detail → assign an existing user to the company with a role.
  • Self-register at /register → lands on "no workspace yet" + verification prompt.
  • HR "add member" → new login emailed credentials, forced reset on first sign-in.
